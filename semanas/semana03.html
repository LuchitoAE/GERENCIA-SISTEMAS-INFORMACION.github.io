Computación en la nube - Semana 3

Explorando la administración, funciones y tecnologías clave para implementar soluciones en la nube en las organizaciones.

1. Administración en la nube ▶
  • Conceptos básicos de administración: La administración en la nube implica la gestión integral de recursos, usuarios y políticas en entornos de computación en la nube. Incluye monitoreo continuo, optimización de costos y aseguramiento de la disponibilidad de servicios para maximizar el rendimiento organizacional.
  • Gestión de identidades y accesos (IAM): Herramientas como AWS IAM o Azure Active Directory permiten controlar quién accede a qué recursos. Se definen roles, permisos y autenticación multifactor (MFA) para prevenir brechas de seguridad y cumplir con regulaciones como GDPR o HIPAA.
  • Monitoreo y logging: Plataformas como CloudWatch (AWS) o Stackdriver (Google Cloud) recopilan métricas en tiempo real sobre uso de CPU, almacenamiento y tráfico de red. Esto facilita la detección de anomalías, alertas automáticas y análisis forense en caso de incidentes.
  • Optimización de costos: Estrategias como el uso de instancias reservadas, autoescalado y análisis de facturación ayudan a reducir gastos. Por ejemplo, herramientas de tagging permiten rastrear costos por departamento o proyecto, evitando sobreprovisionamiento.
  • Gestión de actualizaciones y parches: Automatización mediante scripts o servicios gestionados (ej. AWS Systems Manager) asegura que los servidores y aplicaciones estén actualizados, minimizando vulnerabilidades sin interrupciones en el servicio.
  • Backup y recuperación de desastres: Políticas de respaldo automatizado en servicios como S3 o Azure Blob Storage, combinadas con planes de DR (Disaster Recovery), garantizan la continuidad del negocio con tiempos de recuperación (RTO) y puntos de recuperación (RPO) definidos.
2. Funciones ▶
  • Funciones serverless (FaaS): Modelos como AWS Lambda, Azure Functions o Google Cloud Functions permiten ejecutar código en respuesta a eventos sin gestionar servidores subyacentes. Esto reduce la complejidad operativa y se cobra solo por el tiempo de ejecución real.
  • Escalabilidad automática: Las funciones en la nube escalan horizontalmente según la demanda, manejando picos de tráfico (ej. durante campañas de marketing) sin intervención manual, lo que mejora la eficiencia y reduce costos en comparación con infraestructuras tradicionales.
  • Integración con eventos y triggers: Se activan por eventos como uploads a almacenamiento (S3), mensajes en colas (SQS) o cambios en bases de datos. Esto habilita arquitecturas event-driven, ideales para microservicios y procesamiento en tiempo real.
  • Desarrollo y despliegue ágil: Soporte para lenguajes como Python, Node.js y Java facilita el desarrollo rápido. Herramientas CI/CD (ej. AWS CodePipeline) permiten despliegues continuos, con pruebas unitarias y entornos de staging para minimizar errores en producción.
  • Seguridad en funciones: Políticas de ejecución en entornos aislados (sandbox), integración con IAM para permisos granulares y cifrado de datos en tránsito y reposo protegen contra inyecciones o accesos no autorizados.
  • Casos de uso comunes: Procesamiento de imágenes, APIs backend, ETL (Extract, Transform, Load) para datos y automatización de tareas como notificaciones push o análisis de logs, todo sin preocuparse por la infraestructura subyacente.
3. Tecnología ▶
  • Infraestructura como Servicio (IaaS): Proporciona recursos virtuales como VMs (ej. EC2 en AWS), redes y almacenamiento. Tecnologías subyacentes incluyen virtualización con hipervisores como KVM o VMware, permitiendo abstracción del hardware físico para mayor flexibilidad.
  • Plataforma como Servicio (PaaS): Entornos de desarrollo completos (ej. Heroku o Google App Engine) que manejan runtime, middleware y OS. Utiliza contenedores Docker y orquestadores como Kubernetes para despliegues portables y escalables.
  • Software como Servicio (SaaS): Aplicaciones listas para usar (ej. Office 365 o Salesforce) accesibles vía web. Backend impulsado por arquitecturas multi-tenant, bases de datos distribuidas como NoSQL (MongoDB) y CDN para entrega global de contenido.
  • Tecnologías de almacenamiento y bases de datos: Opciones como bloques (EBS), objetos (S3) y archivos (EFS), junto con DB relacionales (RDS) y no relacionales (DynamoDB). Soporte para big data con herramientas como Hadoop o Spark en la nube para procesamiento distribuido.
  • Redes y seguridad en la nube: VPC (Virtual Private Clouds) para aislamiento, firewalls, VPN y servicios como WAF (Web Application Firewall). Tecnologías como zero-trust y cifrado end-to-end aseguran la confidencialidad en entornos híbridos o multi-nube.
  • Inteligencia Artificial y Machine Learning: Servicios como SageMaker (AWS) o Vertex AI (Google) integran GPUs/TPUs para entrenamiento de modelos. Frameworks como TensorFlow o PyTorch se ejecutan en la nube, habilitando aplicaciones de IA escalables sin hardware especializado local.
